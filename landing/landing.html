Easy Installation

Get Flomark up and running in minutes with automated scripts or manual setup

One-Command Installation

Recommended

The unified installation script automatically sets up everything you need. Choose between Nginx or Apache, optionally enable demo mode, and create your owner account.

Terminal 
# 1. Clone the repository
git clone https://github.com/cli1337/flomark.git
cd flomark

# 2. Configure environment
cd backend && cp env.example .env && nano .env && cd ..

# 3. Run unified installation script
chmod +x install.sh
sudo ./install.sh yourdomain.com
✓ Auto-detects OS
✓ Chooses web server
✓ Creates owner account
✓ Configures SSL-ready

Nginx Installation

Optimized for Nginx web server. Best performance for production deployments.

Terminal 
chmod +x install-nginx.sh
sudo ./install-nginx.sh yourdomain.com

Apache Installation

For Apache/httpd users. Compatible with RHEL, CentOS, and Debian systems.

Terminal 
chmod +x install-apache.sh
sudo ./install-apache.sh yourdomain.com

What Gets Installed?

1

Web Server Setup

Installs and configures Nginx or Apache with reverse proxy, WebSocket support, and security headers

2

Backend Configuration

Sets up Node.js, PM2 process manager, installs dependencies, and runs database migrations

3

Frontend Build

Builds optimized production bundle with Vite, minified assets, and code splitting

4

Owner Account

Interactive prompts to create your owner account with first/last name, email, and password

Prerequisites

  • Node.js 18+ installed
  • MongoDB 6+ running
  • pnpm or npm package manager
  • Nginx or Apache web server
  • Root/sudo access to your server

Step 1: Clone and Configure

Terminal 
git clone https://github.com/cli1337/flomark.git
cd flomark/backend
cp env.example .env
nano .env  # Configure DATABASE_URL, JWT_SECRET, BACKEND_URL

Step 2: Install Dependencies

Backend 
cd backend
pnpm install
npx prisma db push
Frontend 
cd ../frontend
pnpm install
pnpm build

Step 3: Create Owner Account

Terminal 
cd backend
node scripts/make-admin.js admin@example.com OWNER
# Follow interactive prompts for name and password

Step 4: Start Backend with PM2

Terminal 
npm install -g pm2
pm2 start src/server.js --name flomark-backend
pm2 save
pm2 startup  # Follow the command output

Step 5: Configure Web Server

Choose your web server:

Create /etc/nginx/sites-available/flomark:

nginx.conf 
server {
    listen 80;
    server_name yourdomain.com;
    root /path/to/flomark/frontend/dist;
    
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
    }
    
    location /socket.io/ {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
Enable & Restart 
sudo ln -s /etc/nginx/sites-available/flomark /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx

Create /etc/apache2/sites-available/flomark.conf:

apache.conf 
<VirtualHost *:80>
    ServerName yourdomain.com
    DocumentRoot /path/to/flomark/frontend/dist
    
    ProxyPass /api http://localhost:3000/api
    ProxyPassReverse /api http://localhost:3000/api
    
    RewriteEngine On
    RewriteCond %{HTTP:Upgrade} =websocket [NC]
    RewriteRule ^/socket.io/(.*)$ ws://localhost:3000/socket.io/$1 [P,L]
</VirtualHost>
Enable Modules & Site 
sudo a2enmod proxy proxy_http proxy_wstunnel rewrite
sudo a2ensite flomark.conf
sudo systemctl restart apache2

Step 6: Enable SSL (Optional but Recommended)

Terminal 
# For Nginx
sudo apt-get install certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com

# For Apache
sudo apt-get install certbot python3-certbot-apache
sudo certbot --apache -d yourdomain.com
