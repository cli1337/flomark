Easy Installation

Quick Installation

Recommended

One command to install Flomark! Supports multiple databases (MongoDB, PostgreSQL, MySQL, SQLite), auto-configures web server, and creates systemd service for auto-start.

✓ Ubuntu/Debian ✓ Multi-Database ✓ npm/pnpm Support
One-Command Installation git 
# Download and run the installer:
curl -sL https://raw.githubusercontent.com/cli1337/flomark/main/quick-install.sh -o /tmp/flomark-install.sh
sudo bash /tmp/flomark-install.sh

# The script will ask you for:
# ✓ Package manager (npm or pnpm)
# ✓ Database type (MongoDB/PostgreSQL/MySQL/SQLite)
# ✓ Database connection string
# ✓ Installation path & domain
# ✓ Web server (Nginx or Apache)
# ✓ Admin credentials
🗄️ Supported Databases: MongoDB • PostgreSQL • MySQL • SQLite
✓ Multi-database support
✓ Auto Prisma schema generation
✓ Web server configuration
✓ Systemd service
✓ npm/pnpm support
✓ Custom ports

Update Existing Installation

Update to the latest version with automatic backup, rollback, and package manager detection.

Update Script 
# Quick update (auto-detects installation):
curl -sL https://raw.githubusercontent.com/cli1337/flomark/main/update.sh -o /tmp/flomark-update.sh
sudo bash /tmp/flomark-update.sh

# Or run from installation directory:
cd /var/www/flomark
git pull
sudo bash update.sh
✓ Auto backup before update
✓ npm/pnpm detection
✓ Preserves .env & storage
✓ Rollback on failure

What Gets Installed?

1

Database Selection

Choose your database: MongoDB, PostgreSQL, MySQL, or SQLite. Auto-installs if using localhost.

2

Package Manager

Detects npm/pnpm, lets you choose if both available. Installs Node.js 20+ if needed.

3

Backend & Database

Generates Prisma schema for your database, creates tables, installs dependencies, sets up systemd

4

Frontend Build

Builds optimized production bundle with Vite (minified, tree-shaken, code-split)

5

Web Server & Admin

Configures Nginx/Apache with proxy & WebSocket support. Creates admin account and you're live!

Prerequisites

  • Node.js 20+ installed
  • Database: MongoDB, PostgreSQL, MySQL, or SQLite
  • npm or pnpm package manager
  • Nginx or Apache web server
  • Root/sudo access to your server

Step 1: Clone and Configure Backend

Terminal 
git clone https://github.com/cli1337/flomark.git
cd flomark/backend
cp env.example .env
nano .env  # Configure DATABASE_URL, JWT_SECRET, PORT

Database examples:
• MongoDB: mongodb://localhost:27017/flomark
• PostgreSQL: postgresql://user:password@localhost:5432/flomark
• MySQL: mysql://user:password@localhost:3306/flomark
• SQLite: file:./flomark.db

Step 2: Install Backend & Setup Database

Backend Installation 
cd backend
pnpm install

# Then setup your database (choose one):

# For MongoDB:
pnpm db:setup:mongodb

# For PostgreSQL:
pnpm db:setup:postgresql

# For MySQL:
pnpm db:setup:mysql

# For SQLite:
pnpm db:setup:sqlite

💡 Tip: These commands automatically copy the correct Prisma schema, generate the client, and create database tables!

Step 3: Build Frontend

Frontend 
cd ../frontend
pnpm install
pnpm build  # Creates dist/ folder with production files

The built files will be in frontend/dist/ - point your web server to this directory

Step 4: Create Admin Account

Terminal 
cd ../backend
node scripts/make-admin.js
# Follow interactive prompts for email, name, and password

Step 5: Create Systemd Service

Terminal 
sudo nano /etc/systemd/system/flomark-backend.service

# Add this content (replace /path/to/flomark with your actual path):
[Unit]
Description=Flomark Backend Server
After=network.target mongod.service

[Service]
Type=simple
User=root
WorkingDirectory=/path/to/flomark/backend
ExecStart=/usr/bin/node src/server.js
Restart=always
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
Enable & Start Service 
sudo systemctl daemon-reload
sudo systemctl enable flomark-backend
sudo systemctl start flomark-backend
sudo systemctl status flomark-backend

Step 6: Configure Web Server

Choose your web server:

Create /etc/nginx/sites-available/flomark:

nginx.conf 
server {
    listen 80;
    server_name yourdomain.com;
    root /path/to/flomark/frontend/dist;
    index index.html;

    # Handle React Router
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API proxy
    location /api {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # Socket.IO proxy
    location /socket.io {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
Enable & Restart 
sudo ln -s /etc/nginx/sites-available/flomark /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx

Create /etc/apache2/sites-available/flomark.conf:

apache.conf 
<VirtualHost *:80>
    ServerName yourdomain.com
    DocumentRoot /path/to/flomark/frontend/dist

    <Directory /path/to/flomark/frontend/dist>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted
        # Handle React Router
        RewriteEngine On
        RewriteBase /
        RewriteRule ^index\.html$ - [L]
        RewriteCond %{REQUEST_FILENAME} !-f
        RewriteCond %{REQUEST_FILENAME} !-d
        RewriteRule . /index.html [L]
    </Directory>

    # Proxy API requests
    ProxyPreserveHost On
    ProxyPass /api http://localhost:5000/api
    ProxyPassReverse /api http://localhost:5000/api

    # WebSocket proxy
    ProxyPass /socket.io http://localhost:5000/socket.io
    ProxyPassReverse /socket.io http://localhost:5000/socket.io

    RewriteEngine on
    RewriteCond %{HTTP:Upgrade} websocket [NC]
    RewriteCond %{HTTP:Connection} upgrade [NC]
    RewriteRule ^/?(.*) "ws://localhost:5000/$1" [P,L]
</VirtualHost>
Enable Modules & Site 
sudo a2enmod proxy proxy_http proxy_wstunnel rewrite
sudo a2ensite flomark.conf
sudo systemctl restart apache2

Step 7: Enable SSL (Optional but Recommended)

Terminal 
# For Nginx
sudo apt-get install certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com

# For Apache
sudo apt-get install certbot python3-certbot-apache
sudo certbot --apache -d yourdomain.com
